Проведение сделок

Назначение подсистемы

Подсистема предназначена для систематизации и анализа сведений о планируемых и фактических сделках с клиентами (план по сделке, текущее сотояние, результат выполнения). При помощи сделок в системе решаются следующие задачи:

  • оценка эффективности работы отдела продаж в целом и каждого из менеджеров;
  • консолидация информации по экземплярам сделок;
  • накопление статистики по качеству работы со сделками;
  • регламентирование работ со сделками;
  • оценка потенциальных объемов по выигранным сделкам и полученной выгоды.

Настройка подсистемы

Работа со сделками (включая отслеживание этапов работы с ними) возможна только после включения функциональных опций Сделки с клиентами и Управление сделками в разделе Администрирование/CRM и продажи. Регистрация первичного спроса клиента возможна, если дополнительно в разделе Администрирование/Маркетинг и планирование включена функциональная опция Фиксировать первичный спрос.

Процесс настройки ведения сделок состоит из следующих этапов:

  • Определение списка видов сделок, применяемых в торговом предприятии. Определяются те виды сделок, которые используются в торговом предприятии.
  • Определение списка этапов процесса продаж. На основании этапов процесса продаж будет формироваться воронка продаж для анализа эффективности проведения сделки.
  • Классификация списка участников сделки. Определение списка ролей участников сделки и контактных лиц участников сделки.
  • Классификация причин проигрыша сделки. Определяется список причин, по которым сделка может быть проиграна.
  • Классификация причин неудовлетворения спроса. Определяется список причин, по которым не был удовлетворен первичный спрос клиента.

Виды сделок
  • Прочие непроцессные сделки - управление сделками не происходит, список этапов для такой сделки не составляется, отследить на каком этапе находится сделка нельзя, можно управлять только статусами сделки
  • Сделки с ручным переходом по этапам - список этапов процессов продаж реггистрируется пользователем, предоставляется возможность управлять продвижением по этапам процессов продаж "вручную", указывая каждый раз, на какой этап перешла работа со сделкой
  • Типовая продажа. Тип сделки определяется названием того бизнес-процесса, с помощью которого производится ее управление. Переход по этапам жестко определяется логикой бизнес-процесса. Бизнес-процесс, с помощью которого происходит управление сделкой, разрабатывается при конфигурировании программы. В качестве примера бизнес-процесса управления сделками в поставку программы включен бизнес-процесс Типовая продажа.

Информация о видах сделок, которые применяются на предприятии вводится в одноименный справочник (раздел Маркетинг и планирование/Настройки и справочники). Справочник Виды сделок с клиентами служит для хранения всех видов сделок, которые могут применяться на предприятии. Это могут быть виды сделок, которые используются для хранения документов в рамках долгосрочных проектов, так и сделки, в рамках которых производится управление процессом продаж в соответствии с указанными этапами продаж. В программе возможно добавление любого количества видов сделок. Для каждого вида сделки указывается тип сделки, в соответствии с которым определяются те процессы продаж, которые могут быть зафиксированы в рамках данной сделки.

Этапы процессов продаж

Список этапов процессов продаж заполняется в одноименном справочнике в разделе Маркетинг и планирование/Настройки и справочники. Использование этапов процессов продажи позволяет отследить работу с клиентом на подготовительном этапе работы с ним и определить на каком этапе клиент отказался работать с нами.

Роли партнеров и контактных лиц

Список ролей, которые могут быть определены для партнеров и контактных лиц, участвующих в сделках, заполняется в соответствующих справочниках Роли партнеров в сделках, проектах и Роли контактных лиц в сделках, проектах. Справочники доступны в пункте меню Маркетинг и планирование/Настройки и справочники. Роли используются при вводе информации о партнерах и контактных лиц, участвующих в сделке. С помощью отчета Окружение сделки можно проанализировать, кто из участвующих в сделке партнеров может повлиять на успешное выполнение сделки.

Причины проигрыша сделок

Причины, по которым может возникать проигрыш сделки, классифицируются с помощью справочника Причины проигрыша сделки. Справочник вызывается в разделе Маркетинг и планирование/Настройки и справочники. Эту информацию в дальнейшем можно использовать для анализа причин проигрыша сделки.

Причины неудовлетворения первичного спроса

При работе со сделками можно фиксировать первичный спрос клиента и затем вводить информацию о том, по какой причине первичный спрос клиента был неудовлетворен. Причины неудовлетворения первичного спроса классифицируются с помощью записей в справочнике Причины неудовлетворения первичного спроса (раздел Маркетинг и планирование/Настройки и справочники).

Регистрация связей между партнерами

Список всех возможных видов связей (клиент-конкурент, знакомый-знакомый и т.д.) регистрируется в справочнике Виды связей партнеров (раздел Нормативно-справочная информация/Настройки и справочники). Информация о взаимосвязях между партнерами вводится в форме партнера (команда Взаимосвязи между партнерами на панели навигации). Информация о связях между партнерами используется при просмотре окружения сделки.

Работа подсистемы

Оформление сделки с клиентом производится в момент обращения клиента с намерением приобрести товар. В сделке может быть зарегистрирован как первичный интерес клиента (список товаров, который он хочет приобрести), так и источник обращения. При работе по сделке менеджер фиксирует информацию о лицах участвующих в сделке и определяет степень их участия (роль партнера или контактного лица).
При заключении сделки менеджер может провести анализ возможных предложений конкурентов по тем товарам, которые хочет закупить клиент. После заключения сделки, все документы, участвующие в процессе продаж, оформляются в рамках конкретной сделки. Менеджер в любой момент может просмотреть список документов, оформленных в рамках сделки, а также список взаимодействий, заданий и файлов, связанных с сделкой.

Новая сделка может быть создана:

  • Из списка сделок (раздел Продажи/Сделки с клиентами).
  • Из списка клиентов.
  • Из карточки клиента.
  • На основании ранее зарегистрированного взаимодействия с клиентом (раздел Органайзер/Взаимодействия).

Работа со сделкой

В процессе работы со сделкой пользователь может выполнять следующие задачи.

  • Составление план-проекта работы со сделкой. Менеджер может сформировать план-проект выполнения сделки и следить за его выполнением.
  • Управление документами сделки. Менеджер может просматривать и оформлять различные документы по сделке, контролировать ход выполнения сделки.
  • Работа с партнерами и клиентами. Менеджер может осуществлять работу с клиентами в рамках сделки: просматривать и изменять окружение сделки, регистрировать взаимодействия с клиентом при проведении сделки (электронные письма, телефонные звонки и т.д.).
  • Регистрация задач (заданий) по сделке. В процессе работы по сделке менеджер может выдавать различные задачи и задания другим сотрудникам предприятия.
  • По сделкам, которые управляются бизнес-процессом, задачи формируются автоматически в соответствии с картой маршрута бизнес-процесса. В то же время при необходимости менеджер может сформировать дополнительные задачи сотрудникам предприятия вручную.
  • По сделкам, которые не управляются бизнес-процессом, менеджер формирует задачи только вручную.
  • Управление процессом продаж по сделке. Если процесс продаж по сделке регулируется бизнес-процессом, то управление процессом продаж по сделке производится автоматически. При ручном управлении сделкой предоставляются дополнительные средства для перехода по этапам сделки, фиксации результатов и т.д.

После завершения работы со сделкой менеджер фиксирует результат работы со сделкой: выиграна или проиграна сделка.

Анализ эффективности сделок

Для осуществления комплексного анализа эффективности работы со сделками в системе предназначены различные отчеты (раздел Продажи/Отчеты по продажам). Отчеты позволяют провести анализ этапов прохождения сделки, начиная с момента возникновения сделки с клиентом (анализ первичного интереса, вероятностный прогноз по сделке), анализа работы по сделке (Воронка продаж) до результатов по завершенным сделкам (Эффективность сделки, Анализ первичного спроса, Анализ причин проигрыша сделки). Дополнительно можно оценить показатели о сравнительной эффективности работы менеджеров с клиентами.
